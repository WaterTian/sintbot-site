# sintbot 平台站重设计 — 设计文档

- 日期：2026-05-19
- 站点：https://sintbot.com （仓库 `WaterTian/sintbot-site`，GitHub Pages）
- 状态：设计已定稿，待转实现计划

## 1. 背景

现有站点是一个 **「cc-bot for Slack」单产品展示站**。但 cc-bot 实际上只是 sintbot
的一个模块 —— 对接 Claude Code 与 IM 的插件。sintbot 要升级为一个**平台**，包含两个
平级的部分：

1. **多模型群聊机器人** —— 把 AI agent 接进飞书/Slack 群。cc-bot 是其中基于
   Claude Code 的一种，已上线；未来同一套机制可接入其他 AI agent。
2. **桌宠** —— 人手一只、趴在桌面的虚拟形象，旁听群聊、关心项目、有性格有情绪地
   陪人工作，并把 cc-bot 的开发状态演成表情动作。

本设计把站点从「单产品站」重设计为「平台站」。

## 2. 目标与非目标

**目标**
- 站点重心从 cc-bot 上移到 sintbot 平台，cc-bot 降级为支柱 ① 的一个成员。
- 新增桌宠（支柱 ②）的完整展示区块。
- 现有打磨过的 cc-bot 深度内容**零改写**保留。
- 维持项目一贯的零构建 / 零依赖 / 单页 / 双语 / 克制 editorial 调性。

**非目标**
- 不实现 cc-bot、桌宠或多模型本身 —— 本设计只改展示站。
- 不改视觉体系（墨黑 / 信号橙 / Fraunces / PCB editorial 保持不变）。
- 不引入构建工具、框架、依赖；不拆多页。

## 3. 关键决策摘要（来自头脑风暴）

| 决策点 | 结论 |
|---|---|
| 平台结构 | 两个平级支柱：① 多模型机器人　② 桌宠 |
| cc-bot 定位 | 支柱 ① 里的「Claude Code 版」，是目前唯一已上线的 agent |
| 成熟度处理 | **平台愿景优先** —— 两支柱当完整愿景并列呈现，不强调谁已上线 |
| 站点结构 | **保持单页长滚动**，`index.html` 不拆分 |
| 视觉调性 | **保持现有克制 editorial**，桌宠形象作点睛，不改整体冷静感 |
| 重构方式 | **方案 A —— 平台外壳 + 内容嵌入**：现有内容外套平台框架，内部基本原样 |
| 桌宠形象 | 复用当前 logo 的 PCB 小生物 |
| 多模型名单 | **全部待定** —— 候选含 Codex、Google(Gemini)、本土模型；站点不写死任何具体型号 |
| 桌宠表情图 | 先用 CSS 微变体占位（路线 1），后续生成专门表情图替换（路线 2） |

## 5. 各区块详细设计

### 5.1 导航

- 品牌：`cc·bot` + `for Slack` 标签 → 改为 **`sint`〔走线动效〕`bot`**。保留现有
  nav PCB 走线动效，锚点从 `cc↔bot` 间移到 `sint↔bot` 间。去掉 `for Slack` 标签。
- 菜单：现有 `场景 / 能力 / 架构 / 接入`（cc-bot 子锚点）→ 改为支柱级
  **`机器人` · `桌宠` · `接入`** + `GitHub` 外链 + 语言切换。
- logo 那只小生物：不变。

### 5.2 平台 Hero

沿用现有 hero 的双栏骨架，内容平台化：

- **eyebrow**：平台级（如 `sintbot · 本地优先 · MIT`）。
- **H1 大标题**：保留现有三行字体处理与错峰揭示动效，仅把第三行换掉：
  - 中文：`群里聊天` / `「Local Agent」` / `桌宠`
  - 英文：`Group chat` / `「Local Agent」` / `Desktop pet`
  - 三行对应平台三切面：在哪（群里聊天）· 是什么（Local Agent）· 新的脸（桌宠）。
- **桌宠形象**：复用 logo 的 PCB 小生物，置于大标题旁，带**克制的纯 CSS idle
  动效**（呼吸 / 眨眼级别的细微浮动），呼应「会动的脸」，不破坏 editorial 冷静感。
- **副标题**：一句话交代 sintbot = 把 AI agent 接进飞书/Slack 群（cc-bot）+
  一只演它状态、陪你工作的桌宠。
- **CTA**：平台级措辞。
- **Slack 对话 mockup**：现有组件原样保留。

### 5.3 「sintbot 是什么」平台引入区

紧接 hero 的新区块，全站唯一把平台讲整的地方。

- kicker：`平台`；标题：一句话定义 sintbot；导语：本地优先 · 接入飞书/Slack ·
  MIT 等平台底色。
- **两张支柱卡并列**，沿用现有 `.card` / `.scenario` 样式（边框、序号、hover），
  不新造组件。每卡一句话 + 一个跳到对应支柱锚点的链接。
  - 支柱 ① 卡：「**多模型群聊机器人** —— 把 AI agent 接进飞书/Slack 群，
    mention 一下就开干。cc-bot 是 Claude Code 版，已上线。」
  - 支柱 ② 卡：「**桌宠** —— 人手一只趴桌面的虚拟形象，旁听群聊、关心项目，
    有性格有情绪陪你工作。」

### 5.4 支柱 ① — 模型层引子

支柱 ① 的开篇区块，夹在平台引入区与现有 cc-bot 内容之间，负责讲清「机器人」是
什么，并让读者理解为什么下面一大段都是 cc-bot。

- kicker：`支柱 ①`；标题：`把 AI agent 接进群`。
- 导语：机器人的活儿是把一个 AI agent 绑进飞书/Slack 群 —— 群友 mention 一下，
  agent 在你本机读意图、干活、回贴。
- **小视觉 · agent 槽位**：沿用现有 chip / pill 样式，不新造组件。
  - `Claude Code → cc-bot · 已上线`（实心 / accent）
  - `其他 AI agent → 规划中`（ghosted 灰显，**不点名任何具体型号**）
- 转场句：`下面是 cc-bot —— Claude Code 版的完整能力。`

**设计要点**
1. 诚实分层：引子明说现在能用的只有 cc-bot，其他 agent 槽位灰显不点名。
2. 不与现有 IMAdapter 区块打架：现有「一个 IMAdapter，所有群聊」讲的是
   *IM 可插拔*（真实已实现）；本引子讲 *agent 可插拔*（**规划中**）。文案上让两者
   形成「两条可插拔的轴」的呼应，但**绝不把 agent 层说成已实现**。
3. 现有 6 个 section 重新编号为 `①·01 … ①·06`。

### 5.5 支柱 ① — 现有 cc-bot 内容（复用）

6 场景 / 本地优先 / 7 能力卡 / IMAdapter / 架构图 / Slack 安装四步：**正文、内联
SVG、i18n 文案全部不动**。唯一改动是各 section 的 kicker 编号（`00–05` →
`①·01–①·06`）。

### 5.6 支柱 ② — 桌宠区块

最大的全新区块。按「**有性格有情绪的伙伴**」定位设计，不是状态显示器。

- kicker：`支柱 ②`；标题：**`给 AI 一个形象。`**
- 导语：cc-bot 在群里干活，但它没有脸。桌宠就是那张脸 —— 一只趴在你桌面的小生物，
  旁听群、关心项目、有情绪地陪你。
- **签名视觉 · 状态板**：桌宠在四种状态下的表情/姿态：
  - `编译中 → 专注`、`部署成功 → 雀跃`、`报错 → 警觉`、`空闲 → 慵懒`
  - **路线 1（先上、占位）**：用 CSS 对 logo 小生物做克制的微变体 —— 改色调、
    小幅姿态位移、极简小配件（汗滴 / 感叹号 / zzz）。零新美术、零依赖。
  - **路线 2（后续升级）**：之后生成 4–5 张专门桌宠表情精灵图替换。状态板**结构上
    预留整图替换位** —— 换图只动 `assets/`，不动 HTML/CSS 结构。
- **内容卡 ×4**（沿用现有 `.card` / `.local` 样式，不新造组件，编号 01–04）：
  - 01 人手一只 —— 每个群友桌面都有自己的一只，同群、各自视角。
  - 02 旁听群聊 —— 实时接飞书/Slack，群里发生什么它都知道。
  - 03 演状态 —— cc-bot 编译 / 部署 / 报错，它用表情动作演出来。
  - 04 有性格 —— 会回应消息、有脾气、有情绪 —— 陪你工作的伙伴。
- 收尾句：`让团队那位 AI 开发同事，有一张会动、会回应的脸。`

### 5.7 CTA / 页脚

- CTA：从 cc-bot 措辞改为平台级措辞（介绍 sintbot 整体，仍指向 GitHub 仓库）。
- 页脚：品牌、tag、导航链接同步改为平台级。

## 6. i18n 改动

所有文案进 `app.js` 的 `translations` 对象，EN + 中文双份。

**新增 key**：平台 hero（eyebrow / 副标题）、「sintbot 是什么」区（kicker / 标题 /
导语 / 两张支柱卡）、模型层引子（kicker / 标题 / 导语 / agent 槽位标签 / 转场句）、
桌宠区（kicker / 标题 / 导语 / 状态板 4 个状态标签 / 4 张卡 / 收尾句）。

**改动 key**：
- H1 第三行：`整个项目` → `桌宠`；`Whole project` → `Desktop pet`。
- 导航各项。
- 章节 kicker 重编号：`00–05` → `①·01–①·06`，新增 `支柱 ②` 等。

**不动**：6 场景 / 本地优先 / 7 能力卡 / IMAdapter / 架构 / 安装 的所有正文 key。

## 7. 技术约束与改动

仍然 **零构建 · 零依赖 · 单页 `index.html` · 双语**。

- `index.html`：新增 平台 Hero 改造、「sintbot 是什么」区、模型层引子、桌宠区的
  结构标记（`data-i18n*`、`data-reveal` 属性）；导航与 meta 改平台级。
- `styles.css`：新增 桌宠区 / 状态板 / 支柱卡 / hero idle 动效 / agent 槽位 的
  样式，全部原生 CSS 自定义属性，沿用现有配色与组件变量。
- `app.js`：只新增 / 改动 i18n 条目。滚动揭示与 section 埋点 observer 按
  `[data-reveal]` / `main section[id]` 自动覆盖新区块，**逻辑无需改**。
- GoatCounter：现有 6 个自定义事件不动；可选新增 2 个事件（桌宠区到达、支柱卡
  点击）。
- `assets/`：路线 1 不加新文件；桌宠表情新图就绪后再放入。
- `<title>` / `description` / og meta 改平台级；`canonical` / `theme-color` /
  `CNAME` 不动。

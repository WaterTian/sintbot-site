00 — Premise

Your team already lives in the group chat. cc-bot meets them there.

Most engineering bots are dashboards in disguise — yet another tab, yet another sign-in. cc-bot is the opposite: it sits inside the same IM channel where your team already coordinates and takes natural-language orders. Compile a preview, query logs, ship a build — Claude Code does the work, your chat thread is the UI.

  • 2IM adapters today — Lark & Slack
  • 3defensive polling layers
  • 0runtime dependencies on Windows

01 — Capabilities

Six muscles, one bot.

  1. 01

    IM adapter abstraction

    One interface, many group chats. The Lark and Slack adapters ship today; a new IM is just another implementation of the same contract — message in, intent out, reply back.

    • Lark
    • Slack
    • plug-in pattern
  2. 02

    Three-layer defensive polling

    IM webhooks lie. cc-bot triangulates against the long-poll channel, a short-poll fallback and a deduplicated emit log, so every message is delivered exactly once even when the network hiccups.

    • long-poll
    • short-poll
    • dedupe ledger
  3. 03

    Multi-agent scheduling

    Slots and tags turn parallel Claude sessions into a polite queue. Two deploys won't fight each other; a query and a deploy can run side by side. Conflicts resolved by config, not chance.

    • slot
    • tag
    • conflict matrix
  4. 04

    HUD status broadcast

    The Claude Code statusline becomes a live indicator: which agents are busy, which slots are free, last message thread. Your terminal shows the same truth the group chat shows.

    • statusline
    • real-time
    • shared state
  5. 05

    Three-tier permission matrix

    Public reads, admin-auto writes, admin-confirm for the scary ones. Each intent is tagged once in the profile; the bot enforces it forever. The chat stays open, the repo stays safe.

    • public
    • admin-auto
    • admin-confirm
  6. 06

    Main-window priority

    A UserPromptSubmit hook lets the developer at the keyboard preempt any background bot task. When you're focused, the bot waits — when you step away, it gets back to work.

    • hook
    • preempt
    • focus-aware

02 — Architecture

A small idea, drawn carefully.

The bot is a thin pipeline. IM messages enter on the left, three polling layers cooperate to make sure none are lost, the scheduler hands them to a Claude Code session under the right permission tier, and the result flows back out — to the chat and to the HUD at the same time.

03 — Install

Three lines to a chat-driven repo.

  1. 01

    Add the marketplace

    # inside Claude Code
/plugin marketplace add WaterTian/cc-bot

  2. 02

    Install the plugin

    /plugin install cc-bot@WaterTian-cc-bot

  3. 03

    Run setup

    /cc-bot:setup

    You'll pick Lark or Slack, paste a few tokens, and cc-bot will auto-detect chat IDs and write .cc-bot/profiles/active.json for you. Then /cc-bot:start brings the bot online.

Slash commands

  • /cc-bot:setup — interactive onboarding
  • /cc-bot:start — bring the bot online
  • /cc-bot:stop — take it offline
  • /cc-bot:new-profile <name> — clone the template
  • /cc-bot:switch <name> — swap active profile
  • /cc-bot:doctor — health check & diagnostics
